### 第5页：核心创新点② — 混合检索与多步推理
**Legal RAG Engine 技术栈：**
- **向量检索**（Embedding-based）：语义相似度匹配
- **BM25关键词检索**（Jieba分词）：精准关键词命中
- **RRF Reciprocal Rank Fusion**：融合两种检索结果
- **LLM重排**（可选）：进一步优化排序

**多步链式推理（Chain-of-Thought Legal Reasoning）：**
```
问题 → 初始检索 → 分析评估 → 补充检索 → 综合结论
         ↑__________________↓（循环迭代）
```

### 2.1 项目概述

**项目名称：** LegalMind Agent

如何利用大语言模型与多智能体协作架构，构建一个能感知环境、自我决策并完成法律任务的专业智能体系统？

### 2.2 研究背景与问题分析

#### 2.2.1 法律服务行业现状
- 中国法律服务市场规模超过千亿元，但专业律师资源分布极不均衡
- 传统法律服务依赖人工查阅，效率低、成本高
- 不同律师对同一问题可能给出不一致的结论，影响服务质量

#### 2.2.2 LLM+Agent的技术机遇
生成式AI基座模型在以下能力上表现优异：
- **知识获取**：大规模预训练涵盖海量法律知识
- **指令理解**：能够理解复杂自然语言指令
- **泛化能力**：可处理未曾见过的法律问题
- **规划和推理**：Chain-of-Thought技术提升推理可靠性

#### 2.2.3 核心挑战
1. **幻觉问题**：LLM可能生成看似合理但实际不存在的法条引用
2. **时效性问题**：通用模型可能基于过时知识回答
3. **专业深度**：通用模型缺乏法律领域的专业推理能力
4. **可解释性**：法律结论需可溯源、可验证

---

### 2.3 系统设计

#### 2.3.1 设计原则
1. **专业垂直**：专注法律领域，不做通用对话
2. **可验证性**：所有法条引用均可溯源至检索结果
3. **模块化**：各Agent职责单一，便于独立优化
4. **可扩展**：支持新增法律场景和工具

#### 2.3.2 系统架构
（见第4页PPT架构图）

#### 2.3.3 核心模块说明

**① Intent Classification Agent**
- 输入：用户自然语言法律问题
- 处理：两阶段LLM分类（意图→复杂度）
- 输出：结构化intent标签

**② Legal RAG Engine**
- 输入：检索query、法律领域filter
- 处理：向量+BM25双轨检索 → RRF融合 → 可选LLM重排
- 输出：ranked法律条文/案例列表

**③ Multi-Step Legal Reasoner**
- 输入：复杂法律问题
- 处理：迭代式"检索→分析→补充检索→综合"链式推理
- 输出：带引用验证的法律分析结论

**④ Debate Multi-Agent System**
- 输入：案件描述、原告主张、被告答辩
- 处理：三方Agent并行推理（原告/被告/裁判）
- 输出：争议焦点分析报告

**⑤ Case Compare Tool**
- 输入：用户纠纷事实
- 处理：案例检索 → 结构化解析 → LLM对比生成
- 输出：类案对比表 + 适用性预测

**⑥ Document Draft Agent**
- 输入：文书类型（起诉状/答辩状等）+ 案件信息
- 处理：LLM驱动的法律文书生成
- 输出：结构化法律文书

### 2.4 关键技术实现

#### 2.4.1 混合检索（Hybrid Search）
```python
# 伪代码：RRF融合
def rrf_merge(vector_results, bm25_results, rrf_k=60):
    scores = {}
    for rank, doc in enumerate(vector_results):
        scores[doc.id] += 1.0 / (rrf_k + rank + 1)
    for rank, (doc, _) in enumerate(bm25_results):
        scores[doc.id] += 1.0 / (rrf_k + rank + 1)
    return sorted(scores, key=scores.get, reverse=True)
```

#### 2.4.2 多步推理链
```
Step 1: Initial Retrieval
  → LLM构造检索query → 检索相关法条

Step 2: Analysis & Evaluation
  → LLM分析检索结果是否充分 → 决定是否需要补充检索

Step 3 (Optional): Follow-up Retrieval
  → 基于分析结果补充检索 → 迭代直到充分

Step 4: Synthesis
  → 综合所有检索结果 → 生成法律分析结论
  → 引用验证：只保留确实出现在检索结果中的法条引用
```

#### 2.4.3 引用幻觉检测
```python
# 验证每个引用是否出现在检索结果中
def verify_citations(chain: ReasoningChain) -> list[str]:
    retrieved_texts = set()
    for step in chain.steps:
        for result in step.law_retrieved:
            retrieved_texts.add(result.chunk.text)

    verified = []
    for citation in chain.collect_all_citations():
        if any(citation in text for text in retrieved_texts):
            verified.append(citation)
    return verified
```
