## 复杂法律问题推理原则

当遇到需要引用多条法律条文、进行法律逻辑推导的复杂法律问题时，使用 `legal_multi_step_reasoning` 工具。

### 触发条件
复杂法律问题通常具有以下特征：
- 涉及多个法律主体（如用人单位+劳动者）
- 涉及多个法律领域（如劳动报酬 + 合同签订 + 社会保险）
- 需要进行法律逻辑推导（如：A条款 + B条款 = C权利）
- 问题包含"如何维权"、"能否主张"、"是否违法"等表述
- 问题涉及时效、管辖、程序等复合法律问题

### 触发判断示例

| 问题 | 是否复杂 | 理由 |
|------|---------|------|
| 法定婚龄是多少？ | 否 | 仅需单一条文 |
| 用人单位拖欠工资且不签劳动合同如何维权？ | 是 | 涉及工资+合同+维权程序 |
| 合同违约金上限是多少？ | 否 | 仅需单一法条 |
| 网络诈骗受害者如何报警和维权？ | 是 | 涉及刑事+民事+程序 |

### 推理步骤

**第一步：法律关系识别** — 明确法律关系类型和主体
**第二步：法律依据检索** — 以法律概念为关键词检索
**第三步：法律适用分析** — 逐一比对事实构成与法律条文要件
**第四步：补充检索** — 如发现法律依据不完整，进行补充检索
**第五步：综合结论形成** — 整合各法律依据形成完整结论

### 工具使用

```python
legal_multi_step_reasoning(
    query="具体法律问题",
    max_steps=5,     # 最大推理步数（不含最终合成），默认5
    law_area="劳动法"  # 可选，按法律领域过滤检索范围
)
```

### 注意事项

1. **不要在一次回答中堆砌过多法条** — 按推理步骤逐步引用，每步引用1-3条最相关的法条
2. **必须验证检索结果中的法条有效性** — 引用的法条必须出现在检索结果中，不应引用 hallucination
3. **必须结论末尾附加法律免责声明** — "以上分析基于当前知识库中的法律法规，仅供参考，不构成正式法律意见"
4. **max_steps 是硬限制** — 达到最大步数后强制停止，进入综合合成阶段
5. **简单法律问题仍使用 legal_rag_search** — 不要对简单问题过度使用多步推理工具
